# Nemotron Ultra Runtime Status - 2026-06-06

This document records the current release state for
`NVIDIA-Nemotron-3-Ultra-550B-A55B-JANGTQ_1L` in vmlx-swift and Osaurus.
It separates confirmed reference rows from Swift runtime diagnostics so the
8 tok/s claim does not get applied to the wrong runtime mode.

## Fixed

  without casting TurboQuant routed-expert norms.
  - Change:
    `Libraries/MLXLMCommon/Load.swift` no longer blanket-skips BF16
    conversion for `weight_format=mxtq` / JANGTQ-native bundles.
  - It converts only affine `.scales` and `.biases` parameters for those
    bundles and explicitly preserves `.tq_norms`, because JANGTQ kernels infer
    their signature from the raw norm dtype.

- Nemotron-H activation dtype retention is explicit in the Swift runtime.
  - Each decoder block casts the residual output back to the incoming
    activation dtype.
  - The final normalized hidden state is cast to the embedding dtype before
    `lm_head` or tied `asLinear`.
  - This mirrors the JANG-side speed diagnosis where accidental float32
    activation widening made `lm_head` and later BF16 paths much slower.

- Hybrid SSM cache topology is source-guarded.
  - Nemotron Ultra cache proof requires both ordinary KV and SSM companion
    state.
  - Legacy KV-only disk payloads are rejected for hybrid cache restore.
  - In-memory SSM keys and disk companion keys use the same model/media-salted
    token key contract.
  - BatchEngine, TokenIterator, and NativeMTP cache-store paths now run
    prompt-and-paged-block SSM rederive for hybrid text prompts even when the
    KV snapshot requires disk-backed restore. Disk-backed hybrid restore is the
    path that needs SSM companion state; it must not fall back to exact-boundary
    snapshot extraction only.
